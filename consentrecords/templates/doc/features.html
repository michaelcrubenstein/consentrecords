  1. public: https://www.pathadvisor.com
    Workflows

    5. View your pathway of previous experiences.

      A pathway is composed of user-entered experiences as well as experiences offered by an organization.

      6. A user-entered experience has the following data:

      1. Organization

        The name of the organization is optional.

      2. Location

        For organizations that have multiple locations, the name of the location at which this experience occurred. The name of the location is optional.

      3. Offering

        The name of the experience. For a program within an organization, it may be the name of the program. For a job, it could be the job title.

      4. Tags
      5. Experience Phase:
        1. Done
        2. Doing Now
        3. Goal
      6. Start Date

        Specifying the start date is optional. When dates are specified, they are specified by a year, a month and an optional day.

        When the Experience Phase is Done, a specified start date is between the user's birthday and the current day (inclusive). If the user's birthday date is not specified, the minimum start date is the first day of the month of the user's birthday.

        When the Experience Phase is Done, the default start date for an experience is one year before the current date with only the year and month specified, but no date specified.

        When the Experience Phase is Doing Now, a specified start date is between the user's birthday and the current day (inclusive). If the user's birthday date is not specified, the minimum start date is the first day of the month of the user's birthday.

        When the Experience Phase is Doing Now, the default start date for an experience is the current date with only the year and month specified, but no day specified.

        When the Experience Phase is Goal, a specified start date is between the current date and the last day of the year 50 years in the future (inclusive).

        When the Experience Phase is Goal, the default start date for an experience is one year after the current date with only the year and month specified, but no date specified.

      7. End Date

        Specifying the end date is optional. When dates are specified, they are specified by a year, a month and an optional day.

        When the Experience Phase is Done or Doing Now, a specified end date is between the experience start date (if specified) and the current day (inclusive). the experience start date is not specified, then the end date is between the user's birthday and the current day (inclusive). If the user's birthday date is not specified, the minimum end date is the first day of the month of the user's birthday.

        When the Experience Phase is Done, the default end date for an experience is the current date with only the year and month specified, but no day specified.

        When the Experience Phase is Done, the default end date for an experience is one year after the current date with only the year and month specified, but no day specified.

        When the Experience Phase is Goal, a specified end date is between the current date and the last day of the year 50 years in the future (inclusive).

        When the Experience Phase is Goal, the default end date for an experience is two years after the current date with only the year and month specified, but no day specified.

Data auditing

  1. Instances and values are write-once.
  2. Instances and values are deleted by writing a record stating when they are deleted.
  3. All writes of instances and values are associated with a transaction that records the user and the timestamp

Value Ordering

  1. Values have a 0-based position as part of their record.
  2. Positions are written at the time the value is written.
  3. No UI for changing positions.
